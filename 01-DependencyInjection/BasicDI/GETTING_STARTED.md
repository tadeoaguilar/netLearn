# Getting Started with BasicDI

## Quick Start Guide

### 1. Navigate to the Project
```bash
cd /Users/tadeo/gitlab/netLearn/01-DependencyInjection/BasicDI/BasicDI
```

### 2. Verify the Project Setup
```bash
dotnet build
```

You should see a successful build message.

### 3. Project Structure

Your project should have this structure:
```
BasicDI/
├── BasicDI.csproj          # Project file with dependencies
├── Program.cs              # Main entry point (you'll edit this)
├── EXERCISE.md            # Step-by-step exercise guide
├── GETTING_STARTED.md     # This file
└── (folders you'll create)
    ├── TightlyCoupled/    # Part 1: Anti-pattern examples
    ├── WithDI/            # Parts 2-6: DI implementation
    └── app.log            # Generated by FileLogger
```

### 4. Follow the Exercise

Open [EXERCISE.md](EXERCISE.md) and follow the step-by-step instructions.

The exercise is divided into 6 parts:
1. **Part 1**: See the problem (tight coupling)
2. **Part 2**: Introduce DI with interfaces
3. **Part 3**: Use the DI container
4. **Part 4**: Swap implementations
5. **Part 5**: Understand service lifetimes
6. **Part 6**: Challenge exercise

### 5. Running Your Code

After each part, run your code to see the results:
```bash
dotnet run
```

### 6. Tips for Success

1. **Create folders as you go**: The exercise mentions files like `TightlyCoupled/UserService.cs`. Create the folders first:
   ```bash
   mkdir TightlyCoupled
   mkdir WithDI
   ```

2. **Create files in your IDE**: Use VSCode or your preferred editor to create the `.cs` files mentioned in the exercise.

3. **Type the code yourself**: Don't copy-paste. Typing helps you learn!

4. **Experiment**: Try changing things to see what breaks and why.

5. **Ask questions**: If something doesn't make sense, that's normal! Think through it or ask for help.

### 7. Common Commands

```bash
# Build the project
dotnet build

# Run the project
dotnet run

# Clean build artifacts
dotnet clean

# Restore dependencies
dotnet restore
```

### 8. Troubleshooting

**Build errors?**
- Make sure you're in the `BasicDI/BasicDI` directory (note: nested folder)
- Run `dotnet restore`

**Namespace errors?**
- Check your namespace matches the folder structure
- Ensure your namespace is `BasicDI.TightlyCoupled` or `BasicDI.WithDI`

**Can't find types?**
- Make sure you have `using` statements at the top of files
- Example: `using BasicDI.WithDI;`

### 9. Need Help?

If you get stuck:
1. Check the error message carefully
2. Review the exercise instructions
3. Check your namespaces and using statements
4. Make sure files are in the correct folders
5. Ask Claude for guidance!
